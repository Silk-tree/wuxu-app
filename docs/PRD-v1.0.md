## 1. 产品概述

### 1.1 核心定位

物序是一款面向城市家庭的全屋食品与日用品智能管理工具。它将家庭中的食品、日用品、药品等物资视为一套需要精细化管理的"微型物流系统"，通过数字化记录、保质期追踪与智能提醒，帮助用户降低浪费、提升家庭库存管理效率。

### 1.2 价值主张

- **减少浪费**: 通过到期日排序和提醒机制，让用户在食品过期前及时处理或使用 [PM hypothesis]
- **心中有数**: 将家庭物资从"散落在各处的模糊记忆"转变为"结构化的数字清单"
- **轻量易用**: 不追求复杂的条码扫描或AI识别，以低门槛的手动录入和直观的视觉反馈为核心体验

### 1.3 目标用户画像

**主要用户群**

| 维度 | 描述 |
|------|------|
| **年龄** | 25–45 岁 |
| **居住状态** | 城市家庭，两人及以上同住 |
| **生活态度** | 注重生活品质，对食品浪费敏感，有整理收纳习惯或意愿 |
| **技术接受度** | 能够熟练使用智能手机，愿意为提升生活效率使用工具类 App |
| **使用场景** | 采购后录入新物品、定期查看即将过期的物品、整理冰箱/储物柜前核对库存 |

**典型用户故事**

> 小李，32 岁，已婚无娃，周末会去超市集中采购一周食材。她经常发现冰箱里有些蔬菜已经腐烂，或是某些调味品过期了几个月。她希望有一个工具，在采购时快速记录保质期，并在快过期时提醒她。

> 老张，40 岁，三口之家，家里常备药品和孩子的零食。他希望清楚知道每种药品的有效期，以及哪些零食快要过期，避免给孩子吃到过期食品。

## 2. 需求背景与市场机会

### 2.1 业务背景与触发源

中国城市家庭每年因食品过期导致的浪费金额可观 [To be confirmed — 需补充行业数据]。当前市场上虽有部分库存管理工具，但普遍存在以下问题：

1. **门槛过高**: 部分产品依赖条码扫描或AI图像识别，对国内商品的覆盖率不足，导致用户录入困难
2. **场景错配**: 企业级库存管理工具（如ERP）功能繁重，不适合家庭场景
3. **体验粗糙**: 现有家庭库存类App界面陈旧，交互体验不符合现代用户预期

物序选择以"手动录入 + 视觉化到期提醒"作为 MVP 切入点，在精确度和易用性之间取得平衡。

### 2.2 用户痛点

| 痛点 | 具体表现 | 后果 |
|------|---------|------|
| **遗忘保质期** | 购买的食品随手放入冰箱/柜中，不记得何时到期 | 食品过期变质，造成浪费和健康风险 |
| **库存模糊** | 不清楚家里有什么、有多少 | 重复购买或临时缺货 |
| **缺乏提醒** | 没有系统性机制提醒即将过期的物品 | 错过最佳使用/处理时机 |
| **管理成本高** | 现有工具录入流程复杂、界面不友好 | 用户难以坚持长期使用 |

### 2.3 竞争分析

| 竞品 | 产品定位 | 核心优势 | 核心劣势 |
|------|---------|---------|---------|
| **过期啦** | 食品保质期管理 | 专注食品场景，支持条码扫描 | 仅支持食品，界面设计较旧，功能单一 |
| **收食** | 家庭食品库存 | 支持多人共享，有社区食谱推荐 | 功能臃肿，非核心功能干扰主线体验 |
| **Excel/备忘录** | 手动记录 | 完全自由，零学习成本 | 无提醒功能，无结构化视图，难以维护 |
| **企业库存App** | 仓储管理 | 功能全面 | 严重错配家庭场景，操作复杂 |

**机会点**

1. **简洁专注**: 不做社交、不做食谱、不做电商，只做"录入—查看—提醒"核心闭环
2. **视觉直观**: 用颜色编码（红/黄/绿）替代纯文字日期，一眼识别紧急程度
3. **声效反馈**: 添加物品时的正向声效，强化录入成就感，提升留存

## 3. 功能需求

### 3.1 功能总览表

| # | 模块 | 功能描述 |
|---|------|---------|
| 1 | 首页/清单页 | 以瀑布流卡片形式展示所有物品，按到期日升序排列。卡片通过红/黄/绿三色背景标识紧急程度。支持左滑删除、右滑编辑。空状态展示友好提示与添加引导 |
| 2 | 添加物品 | 表单录入页面，包含名称（必填）、分类选择器（食品/日用品/药品/其他）、保质期日期选择器（必填）、数量（数字输入）、存放位置（文本输入）。提交后播放"咔哒"声效反馈 |
| 3 | 物品详情 | 展示物品全部信息（名称、分类、保质期、数量、位置、添加时间），提供编辑入口和删除操作。过期物品卡片带呼吸动画 |
| 4 | 设置页 | 包含通知开关（开启/关闭过期提醒）、付费入口（查看当前存储量与上限，解锁无限量）、关于应用信息 |
| 5 | 本地推送通知 | 系统在物品到期前（默认 T-3 天、T-1 天、当天）触发本地推送提醒。用户可在设置中关闭 |
| 6 | 付费解锁 | 免费用户最多存储 20 件物品，达到上限后需支付 1 元解锁无限量存储。付费状态本地持久化 |

```
|  | 🍞 全麦面包        2026-06-28      |  |  <- 黄色卡片（7天内过期）
|  | 食品 · 厨房柜 · 2袋                |  |
|  | 还有 3 天过期                      |  |
|  +------------------------------------+  |
|                                          |
|  +------------------------------------+  |
|  | 🧴 洗发水          2026-09-15      |  |  <- 绿色卡片（安全）
|  | 日用品 · 浴室 · 1瓶                |  |
|  | 还有 82 天过期                     |  |

|                                          |
|           [空状态插图]                   |
```

#### 3.2.2 功能描述

**业务逻辑**

- 进入首页时，系统从本地数据库加载所有物品，按保质期到期日升序排列（最近到期的排在最前）
- 卡片颜色规则：
  - **红色**: 当前日期 > 保质期日期（已过期）
  - **黄色**: 保质期日期 - 当前日期 ≤ 7 天（即将过期）
  - **绿色**: 保质期日期 - 当前日期 > 7 天（安全）
- 红色过期卡片带有"呼吸动画"（透明度在 0.7–1.0 之间循环，周期 2 秒），吸引用户注意
- 数量展示规则：数量为 1 时不显示数量，大于 1 时显示"x{N}"

**交互逻辑**

- 点击卡片 → 跳转物品详情页
- 左滑卡片 → 露出删除按钮（红色背景，白色垃圾桶图标），点击确认删除
- 右滑卡片 → 露出编辑按钮（蓝色背景，白色铅笔图标），点击跳转编辑页
- 点击右上角"+"图标 → 跳转添加物品页
- 点击搜索图标（V1.0 预留入口，可点击但提示"搜索功能即将上线"）
- 底部导航：首页（高亮）、添加、设置

**边界与异常**

- 空状态：当物品数量为 0 时，隐藏列表，展示空状态插图和引导文案
- 数据加载失败：展示"加载失败，点击重试"提示，点击后重新加载
- 列表超过 50 项：保持流畅滚动（Flutter ListView.builder 自动回收）

```
|  | [食品] [日用品] [药品] [其他]      |  |  <- 分类选择器

|                                          |
|  保质期 *                                |
|  +------------------------------------+  |
|  | 2026-07-15  ▼                      |  |  <- 日期选择器
|  +------------------------------------+  |
|                                          |
|  数量                                    |
|  +------------------------------------+  |
|  | 1                                  |  |  <- 数字输入，默认1
|  +------------------------------------+  |
|                                          |
|  存放位置                                |
|  +------------------------------------+  |
|  | 如：冰箱冷藏室                     |  |  <- 文本输入框
|  +------------------------------------+  |
|                                          |
|  [❗ 免费存储还剩 5 个名额]              |  <- 付费提示（达到上限时显示）
|                                          |
|  +------------------------------------+  |
|  |           保  存                   |  |  <- 主按钮
|  +------------------------------------+  |
|                                          |
+------------------------------------------+
```

#### 3.3.2 功能描述

**业务逻辑**

- 名称字段：最大 50 个字符，必填，首尾空格自动去除
- 分类字段：单选，默认"食品"，选项为食品/日用品/药品/其他
- 保质期字段：日期选择器，必填，默认当天日期，最小可选日期为当天，最大可选日期为当天+10年
- 数量字段：整数，范围 1–999，默认 1
- 存放位置字段：最大 100 个字符，可选，提供常见位置快捷标签（冰箱、冷藏室、厨房柜、浴室、卧室、客厅）
- 保存时校验：名称非空且去除空格后长度 > 0，保质期已选择
- 保存成功：播放"咔哒"声效（清脆的机械快门声，时长约 0.3 秒），返回首页并刷新列表
- 免费用户达到 20 件上限时：保存按钮置灰不可点击，提示"免费存储已达上限，1 元解锁无限量"

**交互逻辑**

- 点击"返回" → 如果表单已填写，弹出"是否放弃编辑？"确认框；未填写则直接返回
- 点击日期字段 → 弹出底部日期选择器（年月日滚轮）
- 点击分类选项 → 选中态切换，单选互斥
- 点击保存 → 校验通过则保存并返回；校验失败则提示具体错误（如"请输入物品名称"）

**边界与异常**

- 名称重复：允许同名物品存在（如用户买了两箱相同的牛奶），通过添加时间区分
- 日期选择器适配：iOS 使用 CupertinoDatePicker，Android 使用 MaterialDatePicker
- 键盘遮挡：当软键盘弹出时，表单自动上滚确保当前输入框可见

## 5. 数据模型

### 5.1 Item（物品）实体

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | String | 主键，UUID v4 | 全局唯一标识 |
| `name` | String | 必填，1–50 字符 | 物品名称 |
| `category` | Enum | 必填 | 分类：`food` / `daily` / `medicine` / `other` |
| `expiryDate` | Date | 必填，≥ 当天 | 保质期日期（存储为 yyyy-MM-dd，无时区） |
| `quantity` | Integer | 必填，1–999，默认 1 | 数量 |
| `location` | String | 可选，0–100 字符 | 存放位置 |
| `createdAt` | DateTime | 必填，自动填充 | 创建时间 |
| `updatedAt` | DateTime | 必填，自动更新 | 最后更新时间 |

### 5.2 UserState（用户状态）实体

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `hasUnlocked` | Boolean | 必填，默认 false | 是否已付费解锁无限量 |
| `notificationEnabled` | Boolean | 必填，默认 true | 通知开关状态 |
| `itemCount` | Integer | 必填，默认 0 | 当前物品数量（缓存，实际以 Item 表计数为准） |

### 5.3 数据关系

```
UserState (1) ──< 全局唯一，本地存储
Item (N) ──< 用户创建的物品列表，本地 SQLite 存储
```

**存储策略**

- V1.0 所有数据存储在本地 SQLite，不上传服务器
- 付费状态使用本地持久化（SharedPreferences / NSUserDefaults）+ 本地加密校验防篡改

## 6. 交互设计要点

### 6.1 声效反馈

- **触发时机**: 用户成功添加物品、保存编辑后
- **音效描述**: 清脆的"咔哒"声，类似机械相机快门或订书机按压声，时长 0.2–0.3 秒
- **技术实现**: Flutter `audioplayers` 播放本地 MP3 资源
- **边界**: 若用户手机静音或音量设为 0，自动静默，不额外提示

### 6.2 列表滑动操作

- **左滑删除**: 滑动距离超过卡片宽度的 30% 时，露出红色删除区域。松手后：
  - 滑动距离 < 30%：卡片回弹，恢复初始状态
  - 滑动距离 ≥ 30%：卡片保持展开状态，展示删除按钮，点击后弹出确认框
- **右滑编辑**: 滑动距离超过卡片宽度的 30% 时，露出蓝色编辑区域。松手后逻辑同上，点击后跳转编辑页
- **动画**: 滑动过程使用 Flutter `Dismissible` 或自定义 `Slidable` 组件，带有弹性回弹效果

### 6.3 空状态设计

- **触发条件**: 物品数量为 0
- **视觉**: 居中的柔和插图（一个空冰箱或空储物柜的线描插画）
- **文案**: "家里空空如也，去添加第一件物品吧"
- **引导**: 文案下方展示"+ 添加物品"主按钮，点击跳转添加页
- **动效**: 插图带有轻微的上下浮动动画（周期 3 秒，幅度 5px），增加页面活力

### 6.4 过期物品呼吸动画

- **触发条件**: 当前日期 > 保质期日期（已过期物品）
- **动画效果**: 卡片背景色透明度在 0.7–1.0 之间循环变化，营造"呼吸"感
- **周期**: 2 秒/次，ease-in-out 缓动
- **性能**: 使用 Flutter `AnimationController` + `Tween`，确保 60fps

### 6.5 状态颜色规范

| 状态 | 颜色 | Hex 值 | 应用场景 |
|------|------|--------|---------|
| 已过期 | 红色 | `#FF4D4F` | 卡片背景、状态标签、推送标题 |
| 即将过期 | 黄色/橙色 | `#FAAD14` | 卡片背景、状态标签 |
| 安全 | 绿色 | `#52C41A` | 卡片背景、状态标签 |

## 7. 非功能需求

### 7.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 冷启动时间 | ≤ 2 秒 | 从点击图标到首页可交互 |
| 列表加载 | ≤ 300ms | 100 条物品以内，从数据库加载到渲染完成 |
| 添加物品保存 | ≤ 500ms | 从点击保存到声效播放、返回首页 |
| 滑动帧率 | ≥ 55fps | 列表滑动时无明显卡顿 |
| 内存占用 | ≤ 150MB | 正常使用场景下的峰值内存 |

### 7.2 安全需求

- **数据本地加密**: SQLite 数据库使用 SQLCipher 加密，密钥由设备唯一标识符派生
- **付费状态防篡改**: 付费状态不仅存储在 SharedPreferences，还需在本地生成校验签名，防止简单修改布尔值绕过付费
- **输入安全**: 所有文本输入框限制最大长度，防止超长输入导致性能问题或存储溢出

### 7.3 离线支持

- V1.0 为纯离线应用，所有功能无需网络连接即可使用
- 唯一需要网络的场景是付费解锁，失败时保留当前状态并允许重试
- 应用启动时若检测到无网络且用户未付费，正常展示所有功能；若用户点击付费入口，提示"需要网络连接以完成支付"

### 7.4 兼容性

| 平台 | 最低版本 | 说明 |
|------|---------|------|
| iOS | iOS 13.0 | 支持 iPhone 及 iPad |
| Android | Android 6.0 (API 23) | 覆盖主流设备 |

### 7.5 可访问性

- 所有图标按钮需配备语义标签（accessibility label），供屏幕阅读器朗读
- 颜色不是唯一信息载体：过期状态除红色背景外，还必须有文字说明（"已过期 X 天"）
- 支持系统字体大小设置（Dynamic Type / 系统字体缩放）

## 附录：用户操作流程示例

### 流程 A：添加第一件物品

1. 用户打开物序，看到空状态页面
2. 阅读"家里空空如也，去添加第一件物品吧"，点击"+ 添加物品"
3. 进入添加页，输入"纯牛奶"
4. 分类保持默认"食品"
5. 点击保质期字段，选择日期"2026-07-15"
6. 数量保持默认"1"
7. 存放位置输入"冰箱冷藏室"（或点击快捷标签"冰箱"）
8. 点击"保存"
9. 听到"咔哒"声效，页面下滑关闭，返回首页
10. 首页展示绿色卡片"纯牛奶"，显示"还有 20 天过期"

### 流程 B：处理即将过期物品

1. 用户在首页看到黄色卡片"全麦面包 · 还有 2 天过期"
2. 用户决定今晚吃掉面包
3. 用户左滑卡片，露出删除按钮
4. 点击删除，弹出确认框
5. 确认删除，卡片从列表移除
6. 若剩余物品数量变为 0，页面切换为空状态

### 流程 C：解锁无限量存储

1. 用户已添加 20 件物品，再次添加第 21 件
2. 填写完表单，点击"保存"
3. 系统提示"免费存储已达上限，1 元解锁无限量"
4. 用户前往设置页，点击"升级到无限量"
5. 弹出支付确认框，确认后调起支付
6. 支付成功，设置页显示"已存储 20 件 · 无限量"
7. 返回添加页，成功保存第 21 件物品
